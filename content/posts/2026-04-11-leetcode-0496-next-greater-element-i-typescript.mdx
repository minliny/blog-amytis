# Next Greater Element I

> 先遍历数组并用哈希表记录已访问元素的位置，再判断目标差值是否已经出现；如果 Map 中存在匹配值就直接返回索引，否则插入当前数字和下标继续遍历，整体查找保持 O(n)。

## 代码区

```ts
function nextGreaterElement(nums1: number[], nums2: number[]): number[] {
  const stack: number[] = [];
  const nextMap = new Map<number, number>();

  for (const num of nums2) {
    while (stack.length > 0 && num > stack[stack.length - 1]) {
      nextMap.set(stack.pop() as number, num);
    }
    stack.push(num);
  }

  return nums1.map((num) => nextMap.get(num) ?? -1);
}
```

## 解题思路

先遍历数组并用哈希表记录已访问元素的位置，再判断目标差值是否已经出现；如果 Map 中存在匹配值就直接返回索引，否则插入当前数字和下标继续遍历，整体查找保持 O(n)。

## 复杂度分析

- 时间复杂度：O(n)
- 空间复杂度：O(n)

## 易错点

- 命中补数后应立即返回，避免后续插入覆盖正确索引
- 存在重复值时要保证返回的是之前保存的下标和当前下标

## 优化点

- 使用 Map 存储数值到索引的映射，避免每次重复查找
